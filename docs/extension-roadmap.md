# Extension Roadmap

## Motion Layer

- refactor action selection into a dispatch table
- group related gait logic into reusable motion sequences
- reserve parameters for speed, repeat count, and posture bias

## Expression Layer

- expand OLED face presets for more visible emotional changes
- pair expression changes with movement state
- separate low-level screen drawing from behavior logic

## Input Layer

- unify Bluetooth and voice command handling
- add a command normalization stage before action execution
- keep room for future offline voice keywords or local AI triggers

## Refactor Direction

An AI-assisted refactor workflow can be useful here for:

- mapping duplicated switch branches
- extracting command constants
- generating cleaner module interfaces
- documenting behavior transitions

## Portfolio Angle

Even without publishing the private derivative firmware itself, a repository like this showcases:

- STM32 embedded system understanding
- serial communication handling
- servo behavior design
- interaction-oriented refactor thinking
